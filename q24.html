24. Fetch API
Fetch API предоставляет интерфейст для получения ресурсов (в том числе по сети). Он является более мощным аналагом XMLHttpRequest с более гибким набором функций.
Fetch описывает определение запроса и ответа как объектов (и другие вещи включающиеся в сетевой обмен: тело, заголовки).
Функия fetch имеет один оязательный аргумент - это путь к данным, которые вы хотите получить. Он возвращает promise, который выдает ответ на вопрос.Promise - это объект, который предоставляет возможность завершения или неудачу синхронной операции, и результат завершенной операции (если получилось). Promise - способ работы с асинхронными сообщениями, который позволяет обрабатывать события по мере их поступления.
Body - это fetch-примесь, которая предостовляет методы, относящиеся к телу запроса/ответа, позволяя определять "тело" запроса/ответа и то, как ответ должен быть обработан.
