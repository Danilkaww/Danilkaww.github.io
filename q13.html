13. Компоненты высших порядков в react
HighOrderComponent (комопнент высшего порядка) - функция, которая принимает компонент и возвращает новый (более лучший) компонент, работают с RClass и FunctionalComponent в Kotlin. Используются для того, чтобы избавиться от повторяющихся или похожих в какой-то мере компонентов. Вариантов создания КВП несколько, можно использовать возможности Kotlin (возможности в виде типизированных классов и функций) и классического ООП (параметризацию классов и функций), либо же у react есть свои возможность, которые называются компоненты высокого уровня. :: - используются для передачи функции (не вызова), возвращает объект типа ReactElement. withDisplayName - компонент высшего порядка в react, имеет несколько аргументов (в примере используется 2 аргумента, 1 аргумент - имя, которое нужно добавить, 2 аргумент - класс, к которому нужно добавить), возвращает доработанный класс с новым именем (в примере), не работает с функциональным компонентом (обойти это можно через JS объект, либо с помощью JS переработать функциональный компонент). !Выделить в ошибку: компоненты (функции) высшего порядка каждый раз "улучшают" компонент, переписыая его!
