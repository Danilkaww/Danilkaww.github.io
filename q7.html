7. Компоненты react
Компонент react - это "кирпичик" из которого react строит web-приложение. Либо же это класс, который позволяет создавать react элементы. Имеет свойство и состояние. В компонент мы передаем все в виде свойств, их нужно прописать заранее (kotlin). Каждый компонент снабжается переменной props. Каждый компонент должен реализовать функцию render - функцию прорисовки себя в том месте, где мы этот компонент создали. При создании объекта получается элемент React в поле type которого записана функция. Создаем из компонента элемент react, child - 2 аргумента: класс и обработчик. Функциональный компонент - это компонент, который не имеет состояния, имени, имеет только свойства, но он обрабатывается react`ом быстрее (что-то среднее между react элементом и компонентом). Для создания также используется функция child, чтобы создать функциональный компонент используем функцию functionalComponent в качестве 1 аргумента (в обычном компоненте это класс). В качестве свойств нужно передать свойства компонента как такового. В теле этой функции используем аналог функции render - "+". 2 аргумент не отличается от 2 аргумента обычного компонента. React перерисовывает только те компоненты, которые точно нужно перерисовать, если компоненты не были изменены в приложении, их react перерисовывать не будет.

ЧТОБЫ СОЗДАТЬ КОМПОНЕНТ
Изначально нужно создать файл в котором мы будем писать компонент, далее мы передаем ему свойства с помощью интерфейса (если таковые имеются). После этого мы инициализируем компонент передавая ему свойства и состояние (в случае с функциональным компонентом только свойства). Далее прописываем тело компонента в зависимости от условия задачи и с помощью функции RBuilder преобразуем компонент в элемент React или отрисовываем его.
